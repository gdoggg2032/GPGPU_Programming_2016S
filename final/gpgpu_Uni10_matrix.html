ABOUT OUT PROJECT


Motivation

Deep learning is now considered one of the most popular field in computer science,
and matrix multiplication is definitely the heart of it. We hope to acclerate the
performance of matrix multiplication and assist Uni10 library.

Problem Domain

Exploit the limited resource and schedule the matrices into GPUs.

OUR DOMAIN OF PROBELM

Scheduling multiplication of matrices

Co-Scheduling of CPU and GPU

Exploit the usable resource on GPU

Project Framework

In our structure, the CPU first read the input. Note that the input reading time doesn't count as performance. After reading the input, the scheduler on GPU will check the available resource on GPUs in order to make sure that we exceed the limit of available resource.
After scheduling the matrices into GPUs, the computation starts. In order to accelerate the computation time, we tend minimize the copy times of matrices, which is done by the scheduler. After the computation finishes, GPU copies the output result, matrix C, into CPU. This is the process of our matrix multiplication.


Experiment Result and Analysis

The following includes: experiment setup, experiment results and andlysis.

Experiment Setup

We test our matrix multiplication by comparing the result with NVIDIA's toolkit cuBLAS. We divide the experiment into three scenario: small, medium and large. The three testcases consist of 2 matrices, with size of 512*512, 4096*4096 and 16384*16384, respectively.

Experiment Result & Analysis

We test out program with different scenarios, and the followings are the result.

We first observe that when the file size is comparitively small, e.g. 512*512, naive cuBLAS is better than our method since we have to get GPU status and make scheduling decision. However, the result is acceptable since both the execution time is rather small (150 and 350 ms respectively).

As for the medium file size, our method is a little better than cuBLAS. And as file size grows, we can easily find that our method out-performs the naive cuBLAS method because we have well-planned scheduler to exploit the available resources. It took the naive cuBLAS method 58399 ms to finish the large matrices' calculation, but only took us 36375 ms. That is, under limited resources, our method can push the limited resources to the limit.
